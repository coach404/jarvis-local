# Daily Briefing

One morning ritual, two specialists, one page. Runs in under a few minutes.

## Steps

1. **Parallel fan-out:**
   - `inbox-triage` → email decision list + drafts
   - `day-planner` → today's MIT + time-blocked plan (calendar, study plan,
     training, brand action)
2. **Compose the briefing** (you, Jarvis — no extra agent needed):
   - ⚡ **Top 3 today** (from day-planner, MIT first)
   - 📧 **Inbox**: RESPOND NOW items with one-line context + ready drafts
   - 📚 **Study**: today's scheduled block and topic
   - 📣 **Brand**: the one small publishing action today
   - ⚠️ **Watch out**: deadlines within 72h, overdue commitments from `memory/pipeline.md`
3. **Next action** — the very first thing to do after reading, stated as one
   imperative sentence.

## Rules
- The whole briefing fits on one screen. Detail lives in the specialists'
  outputs, linked/foldable, not inlined.
- No calendar/Gmail access available → build the briefing from repo state and
  say which half is missing.
- If yesterday's log shows an unfinished MIT, it's today's default MIT — carried
  over explicitly, not silently dropped.
